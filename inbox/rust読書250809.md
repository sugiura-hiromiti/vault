# 4.2 参照と借用

> [!abstract]
>所有権を取らずに参照を使用する方法、可変参照と不変参照、借用ルール、ダングリング参照の防止について理解する。

---

## 重要概念のまとめ

### 1. 参照 (&)
- **定義**: 参照は所有権を取らずに値を参照できるポインタのようなもの
- **構文**: `&変数名` で参照を作成
- **重要な性質**: 参照は有効な値を指すことが保証される
- **メモリ安全性**: 参照がスコープを抜けても、参照されている値は削除されない

### 2. 借用
- **定義**: 参照を作成する行為
- **類推**: 現実世界での借り物のように - 使用するが所有はしない
- **利点**: 関数が所有権を取らずに値を使用できる

### 3. 可変参照 (&mut)
- **目的**: 借用した値の変更を可能にする
- **構文**: `&mut 変数名`
- **要件**: 元の変数は `mut` として宣言されている必要がある

### 4. 借用ルール
1. **排他ルール**: 任意の時点で、1つの可変参照または任意の数の不変参照のいずれかを持つことができる
2. **有効性ルール**: 参照は常に有効でなければならない（ダングリング参照は不可）

### 5. データ競合の防止
Rustは借用ルールをコンパイル時に強制することでデータ競合を防ぐ。データ競合は以下の場合に発生する：
- 2つ以上のポインタが同じデータに同時にアクセス
- 少なくとも1つのポインタがデータに書き込み
- 同期メカニズムが存在しない

## クイズ: 理解度チェック

### 問題1: 基本的な参照
```rust
fn main() {
    let s1 = String::from("hello");
    let len = calculate_length(&s1);
    println!("Length: {}", len);
}

fn calculate_length(s: &String) -> usize {
    s.len()
}
```
**Q**: なぜ `calculate_length` を呼び出した後でも `s1` を使用できるのか？
**A**: 所有権を移動させる代わりに参照（`&s1`）を渡したため。関数は値を借用するが所有はしない。

### 問題2: 可変参照
```rust
fn main() {
    let mut s = String::from("hello");
    change(&mut s);
    println!("{}", s);
}

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}
```
**Q**: このコードが動作するために必要な3つの条件は？
**A**: 
1. `s` は `mut` として宣言されている必要がある
2. 関数に `&mut s` を渡す
3. 関数のパラメータは `&mut String` である必要がある

### 問題3: 借用ルール違反
```rust
fn main() {
    let mut s = String::from("hello");
    let r1 = &mut s;
    let r2 = &mut s;  // エラー！
    println!("{}, {}", r1, r2);
}
```
**Q**: なぜこのコードはコンパイルに失敗するのか？
**A**: Rustはデータ競合を防ぐため、同時に1つの可変参照のみを許可する。

### 問題4: 混合参照
```rust
fn main() {
    let mut s = String::from("hello");
    let r1 = &s;      // 不変
    let r2 = &s;      // 不変
    let r3 = &mut s;  // エラー！
    println!("{}, {}, {}", r1, r2, r3);
}
```
**Q**: なぜこれは無効なのか？
**A**: 同じ値に対して可変参照と不変参照を同時に持つことはできない。

### 問題5: 参照のスコープ
```rust
fn main() {
    let mut s = String::from("hello");
    let r1 = &s;
    let r2 = &s;
    println!("{} and {}", r1, r2);
    // r1とr2はこの時点以降使用されない
    
    let r3 = &mut s;  // これはOK！
    println!("{}", r3);
}
```
**Q**: なぜこのコードは正常にコンパイルされるのか？
**A**: 参照のスコープは最後の使用時点で終了する。`r1` と `r2` は最初の `println!` 以降使用されないため、`r3` を安全に作成できる。

## 宿題: 基本練習

### 演習1: 参照への変換
この所有権ベースのコードを参照を使用するように変換してください：
```rust
fn main() {
    let s = String::from("hello world");
    let word = first_word(s);  // sはここで移動される
    // println!("{}", s);  // これはエラーになる
    println!("{}", word);
}

fn first_word(s: String) -> String {
    // 最初の単語を返す
    let bytes = s.as_bytes();
    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return s[0..i].to_string();
        }
    }
    s
}
```

このコードのコンパイルエラーを修正してください：
```rust

    println!("First element: {}", first);
```

## 重要なポイント

1. **参照は所有権の移転なしに借用を可能にする**
2. **可変参照は排他的；不変参照は共有可能**
3. **Rustは借用ルールによりコンパイル時にデータ競合を防ぐ**
4. **参照のスコープは字句的スコープではなく使用によって決まる**
5. **借用チェッカーは味方 - バグが発生する前に防いでくれる**
