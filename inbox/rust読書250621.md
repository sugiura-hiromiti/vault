3. **穴埋め問題**: `io`モジュールをスコープに持ち込むには、`use ______;`と書きます。

4. **選択問題**: `read_line()`は何を返しますか？
   a) String
   b) Result<String, Error>
   c) Result<usize, Error>

### 中級概念

6. **コード分析**: このコードの何が間違っていますか？
   ```rust
   let secret_number = 42;
   let guess = "50";
   match guess.cmp(&secret_number) {
       // ... アーム
   }
   ```

7. **選択問題**: `Err(_) => continue,`の`_`は何を表していますか？
   a) アンダースコアという名前の変数
   b) 値を無視するキャッチオールパターン

   d) 将来のコードのプレースホルダー

8. **説明問題**: 数当てゲームで`guess.parse()`ではなく`guess.trim().parse()`を使用する理由は？

9. **パターンマッチング**: このmatch式を完成させてください：
   ```rust
   match guess.cmp(&secret_number) {
       Ordering::Less => ________,

           println!("正解です！");
   ```

## 解答（クイズ）

### 基本概念
1. b) Cargo.tomlとsrc/main.rsを含む完全なプロジェクト
2. 偽（変数はデフォルトで不変）
3. `std::io`
4. c) Result<usize, Error>（読み取ったバイト数を返す）
5. 最初は不変変数を作成し、2番目は後で変更可能な可変変数を作成します。

### 中級概念
6. 型の不一致：`secret_number`は整数、`guess`は文字列。直接比較できません。
7. b) 値を無視するキャッチオールパターン
8. `trim()`はユーザーがEnterを押したときに追加される改行文字を含む空白文字を削除します。
9. 
   ```rust
   match guess.cmp(&secret_number) {
       Ordering::Less => println!("小さすぎます！"),
       Ordering::Greater => println!("大きすぎます！"),
       Ordering::Equal => {
           println!("正解です！");
           break;
       }
   }
   ```

### 高度な理解
10. 異なるエラーケースを異なって処理したい場合、またはpanicではなく実行を継続したい場合に`match`を使用します。`expect()`はより簡単ですが柔軟性に欠けます。

11. `cargo update`は依存関係を最新の互換バージョンに更新し、Cargo.lockを更新します。`cargo build`は依存関係が変更されていない限り、既存のCargo.lockバージョンを使用します。

12. `parse()`は異なる数値型を返すことができるため、Rustはパースしたい具体的な型を知る必要があります。
